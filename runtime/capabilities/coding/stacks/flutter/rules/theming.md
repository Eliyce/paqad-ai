# Flutter Theming

- Keep color, typography, spacing, and elevation tokens in shared theme layers.
- Do not scatter raw colors, ad hoc spacing values, or feature-specific typography overrides across widgets.
- Prefer semantic theme access through shared tokens and `ThemeData`/`ColorScheme`.
- Support light/dark behavior consistently when the project already does so.
- Document theme-affecting changes in component, screen, and state docs when UI behavior changes.
