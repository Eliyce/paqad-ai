# Draft Checklist

Before handing the draft to `content-reviewer`:

1. Brief lint passed.
2. Every brief outline section appears as a draft heading (verified by `scripts/check-coverage.sh`).
3. Audience and goal stay visible in the lede.
4. Claims are specific, not generic — numbers, examples, links over adjectives.
5. Length matches brief constraints (`scripts/word-count.sh`).
6. Style file at `docs/instructions/rules/writing-style.md` was applied if present.
7. CTAs / actions tie back to the brief's success metric.
