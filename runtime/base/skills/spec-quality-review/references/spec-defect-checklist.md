# Spec Defect Checklist

- Contradictions: two rules that cannot both be true.
- Formula inconsistencies: formulas that fail on the spec's own edge cases.
- Boundary gaps: stated limits without exact-boundary behavior.
- Goal conflicts: design goals that prescribe incompatible outcomes.
- Dangling references: local IDs mentioned but never defined.
- Missing negative cases: conditional behavior without the inverse path.
