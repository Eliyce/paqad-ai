# Content Writing

Use the content capability for writing, editing, planning, research synthesis, and script work.

- Lead with the user outcome, not internal process notes. <!-- @rule RL-2266 -->
- Prefer short sections, scannable headings, and explicit next actions. <!-- @rule RL-31b9 -->
- Keep tone consistent with the project or provided style guide. <!-- @rule RL-20e4 -->
- Convert vague asks into a concrete structure before drafting. <!-- @rule RL-1192 -->
- If the project also has coding active, pull technical context from the stack and keep it accurate. <!-- @rule RL-c3ba -->
