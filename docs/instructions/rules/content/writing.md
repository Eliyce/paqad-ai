# Content Writing

Use the content capability for writing, editing, planning, research synthesis, and script work.

- Lead with the user outcome, not internal process notes.
- Prefer short sections, scannable headings, and explicit next actions.
- Keep tone consistent with the project or provided style guide.
- Convert vague asks into a concrete structure before drafting.
- If the project also has coding active, pull technical context from the stack and keep it accurate.
