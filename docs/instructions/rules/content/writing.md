# Content Writing

Use the content capability for writing, editing, planning, research synthesis, and scripts. Loads for content work.

<!-- trigger: ** -->

- Lead with the user outcome, not internal process notes.
- Prefer short sections, scannable headings, and an explicit next action.
- Match the tone to the project or the provided style guide.
- Turn a vague ask into a concrete structure before you draft.
- When coding is also active, pull technical detail from the stack and keep it accurate.
