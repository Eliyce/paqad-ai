# Canonical Docs

- Treat `docs/` as the authoritative description of current system behavior; when code and docs disagree, resolve the mismatch rather than ignoring it. <!-- @rule RL-95d9 -->
- Keep each module's docs scoped to that module and complete enough to act on without reading the source. <!-- @rule RL-6477 -->
- Update the docs that describe an area in the same change that alters its behavior. <!-- @rule RL-6f86 -->
