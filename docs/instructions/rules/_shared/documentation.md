# Documentation

- Keep `docs/` consistent with the code: update the affected module, flow, API, and error docs in the same change that alters behavior.
- Validate the real stack from manifests (`package.json`, `composer.json`, and equivalents) before documenting it; fix stale stack assumptions first.
- Document specs, flows, API contracts, integrations, and error cases together so they don't drift apart.
- Keep glossary and domain terms consistent with the language used in the code.
