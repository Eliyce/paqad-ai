# Onboarded-Project Overview Maintenance

- [`docs/modules/onboarded-project-overview.md`](../../../modules/onboarded-project-overview.md) is the canonical map of how the framework behaves inside an onboarded project; keep it accurate whenever the *shape* of the system changes (not for every implementation detail). <!-- @rule RL-13b7 -->
- Update it in the same change that: adds, removes, or renames a stage in `docs/instructions/workflows/feature-development.yaml`; adds, removes, or renames a skill under `runtime/**/skills/` or an agent under `runtime/**/agents/`; changes which capability gates a skill or agent; or adds or changes a whole-codebase-consciousness engine (module-map, traceability, cross-module-impact-scanner, RAG/context, verification gates, decision-pause, evidence ledger). <!-- @rule RL-9566 -->
- It is a map, not a copy: point to the authoritative code and per-module docs rather than restating them; do not let it drift into duplicating implementation. <!-- @rule RL-fdd5 -->
- The single source of truth for module slugs and source paths remains `docs/instructions/rules/module-map.yml`; when the overview and the module-map disagree, fix the overview. <!-- @rule RL-e033 -->
